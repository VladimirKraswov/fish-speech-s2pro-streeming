## Почему именно такие параметры

Безопасный профиль под 5090:

* `int8` checkpoint по умолчанию сильно снижает VRAM-давление и даёт запас под `compile + warmup + streaming`
* `bfloat16` оставлен, потому что для 5090 это хороший баланс скорости и стабильности
* `cache_max_seq_len = 768` — не слишком жирный KV cache, но и не тесный для обычного референса + текущего чанка
* `max_new_tokens_cap = 160` — меньше риск обрыва фразы, чем на 128, при этом без ненужного раздувания
* `cleanup_every_n_requests = 24` — не бьёт по каждой заявке, но периодически чистит фрагментацию
* `initial_stream_chunk_size = 10`, `stream_chunk_size = 8` — первый звук приходит быстро, но старт не слишком “мелко нарезан”
* `target_emit_bytes = 6144` и `start_buffer_ms = 120` — стрим остаётся быстрым, но заметно меньше шанс на рывки на старте
* `commit.first` и `commit.next` стали чуть аккуратнее: первый ответ начинается быстро, а дальше чанки крупнее, чтобы меньше рвало интонацию
